## 2. 확정된 결정사항 (되돌리지 말 것)

### ⚠️ 터치 드래그 아키텍처 — 3가지 규칙, 하나라도 되돌리면 버그 재발함

1. **`draggable` 속성은 절대 마크업에 하드코딩하지 말 것.** 반드시 `mousedown`이 드래그 핸들(`.drag-handle`/`.ver-drag-handle`) 위에서 발생했을 때만 JS로 `row.draggable = true` 동적 부여, 드래그 종료 시 `false`로 복원. (이유: 마크업에 `draggable="true"`를 박아두면 터치 기기에서도 행이 네이티브 HTML5 DnD 대상이 되어, 커스텀 터치 핸들러와 경합 — "첫 시도만 되고 이후 안 됨"이라는 flaky 버그의 정체였음. index.js는 원래부터 이 패턴이라 안 걸렸고, versions.js는 하드코딩이라 걸렸었음. `a9216bf`에서 versions.js도 index.js 패턴으로 통일함.)

2. **터치 드롭 위치 판정은 `elementFromPoint`가 아니라 좌표 기반 기하 판정(`resolveTouchDrop`)을 쓸 것.** `elementFromPoint`는 손 뗀 지점 바로 아래 픽셀만 보는데, 헤더(th)나 행 사이 여백에 손을 떼면 tr을 못 찾아 드롭이 통째로 스킵됨(특히 "맨 위로 옮기기"가 구조적으로 항상 실패). `resolveTouchDrop(clientY)`는 draggable 후보 행들을 순회하며 손가락 Y좌표와 각 행 중점을 비교해 삽입 위치를 직접 계산 — 픽셀 아래 요소가 뭐든 상관없음. `touchmove`(인디케이터)와 `touchend`(실제 이동) 둘 다 반드시 같은 함수를 써서 인디케이터=결과 일치를 유지할 것. (`1b2f057`, `807ff14`)

3. **`touchend` 핸들러 맨 앞에 `e.preventDefault()` 유지할 것.** 안 하면 터치 종료 후 브라우저가 같은 좌표에 합성(유령) click을 발생시켜, 만약 그 좌표가 클릭 핸들러 붙은 요소(예: 정렬 가능한 th) 위라면 그 핸들러가 실행되어 드래그 결과를 덮어씀. versions.html에서 실제로 이 버그가 났었음(`6cb4884`). index.html은 현재 tbody 위에 클릭 핸들러 붙은 헤더가 없어 증상은 없지만 동일하게 방어 처리해둠 — 나중에 정렬 헤더 같은 게 추가되면 재발 방지.

- `setupDragDrop`의 tbody 레벨 리스너(`dragover`/`drop`/`touch*`)는 `tbody.dataset.dndBound` 가드로 **최초 1회만 부착**(재렌더마다 중복 부착 방지). 행별 리스너(`dragstart`/`dragend`)는 tbody 위임 방식으로 통일(versions.js `a9216bf`에서 index.js와 동일하게 맞춤).

### GAS_URL — hostname 자동 선택 (2026-07-11부터, 구 규칙 폐기)
- `js/api.js`: `PROD_GAS_URL`/`DEV_GAS_URL` 상수 + `GAS_URL = (location.hostname === 'hmdqa20.github.io') ? PROD : DEV`.
- **더 이상 브랜치별로 GAS_URL 값을 다르게 관리하지 않음** — master/dev의 api.js는 항상 동일 내용. 머지 시 GAS_URL 확인 불필요(과거 "머지 시 git diff로 확인" 규칙 폐기됨, CLAUDE.md에도 반영됨).

### detail.html 표시모드/편집모드 분리
- 기존 티켓 조회 시 기본은 **표시(읽기전용) 모드** — 잠금은 "수정" 버튼 클릭 시에만 `lockTicket()` 호출.
- "저장" 클릭 시 목록으로 이동하지 않고 **그 자리에서 표시모드로 전환**(신규 등록도 동일 — 저장된 row_id로 그대로 표시모드 전환). "티켓등록" 버튼은 표시모드에서만 활성(편집 중엔 비활성 — 데이터 유실/잠금 꼬임 방지).
- 확인결과/비고 필드: 표시모드에서 URL 자동 하이퍼링크(`https?://` 및 `www.` 둘 다 인식, `www.`는 href에 `https://` 자동 추가).

- PowerShell here-string(`@'...'@`)은 `if(){}` 중첩 블록 안에서 깨질 수 있음 → 멀티라인 커밋 메시지는 Bash 툴로 커밋.

- puppeteer 등 헤드리스 브라우저가 로컬에 없음 — CSS/레이아웃 변경은 코드 정합성(브레이스 균형, 우선순위 계산)으로 검증하고 실기기/실브라우저 육안 확인은 사용자가 진행.
