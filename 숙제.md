## 2. 확정된 결정사항 (되돌리지 말 것)

### 실시순서(priority) 정렬 규칙
- 1차: 번호 오름차순(1,2,3…). 번호 있는 항목이 항상 위.
- 2차: **둘 다 빈칸이면 티켓번호(XAX2-NNNN 끝 숫자) 오름차순** (`sortByPriority` + `ticketNo` 헬퍼).

### 드래그(핸들) 4케이스 규칙 — `dragend`
드롭 후 DOM에서 드래그 항목의 **이웃**으로 번호영역/빈칸영역 판정
(`inNumberedZone = 바로 뒤가 번호항목 || (맨 끝이고 바로 앞이 번호항목)`):
1. **번호 → 번호 재배치**: 번호 항목들 DOM 순서로 1..n 재번호.
2. **빈칸 → 번호영역**: 드래그 항목이 번호 획득 + 이후 cascade(재번호로 일괄 처리). 예) 빈칸을 6 앞에 → 빈칸이 6, 기존 6→7…
3. **번호 → 빈칸영역**: 드래그 항목 번호만 삭제, **원래 번호 자리는 gap으로 유지**(나머지 재번호 안 함). 예) `1 2 3 4 5` 에서 2를 빈칸으로 → `1 3 4 5`.
4. **빈칸 → 빈칸**: 변화 없음(재렌더 시 티켓번호순으로 복귀).
- 번호 항목을 드래그로 빈칸구역 최상단 경계에 놓으면 "삭제(빈칸화)"로 해석됨 — 의도된 동작.
- **자물쇠(잠금) 항목은 여전히 드래그/재번호 대상 제외**(`.draggable-row` 없음). 단 드롭다운 cascade 시 자물쇠 항목 처리 규칙은 **미정(아래 TODO 참조)**.

### 헤더 필터 UI (Google식)
- 활성 여부를 **아이콘 교체**로 표시(높이 변화 없음, UI 밀림 없음): 비활성=얇은 ▼(text-secondary/0.9), 활성=**깔때기 SVG + 인디고(`--filter-active #4f46e5`)** + 라벨 볼드/강조색 + 은은한 pill 배경.
- 하단 뱃지(값+×) 방식 **폐지**. 필터 값은 `select`의 `title` 툴팁으로 확인.
- 필터 **해제**는 드롭다운에서 **첫 항목 '전체'(value='') 선택** → 기존 change 핸들러가 처리. 별도 × 버튼/핸들러 없음(제거됨).
- 필터 첫 항목 라벨 = `t('filter_all')` (ko 전체 / jp 全体 / en All / vi Tất cả).
- 컬럼명은 항상 유지(값으로 스왑하지 않음). `syncFilterWrap` 헬퍼가 동적 필터(담당자·확인버전)의 라벨/아이콘/툴팁 동기화.
- SVG 상수: `CHEVRON_SVG`/`FUNNEL_SVG`, `setFilterIcon(wrapEl, active)`.

### 섹션 자동 접힘
- 판정은 **원본 개수**(`allTickets[group].length`) 기준 — 필터/검색 결과가 0개여도 그룹은 열린 채 "항목 없음" 표시. 그룹에 티켓이 애초에 하나도 없을 때만 접힘.
