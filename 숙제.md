## 2. 확정된 결정사항 (되돌리지 말 것)

### 실시순서(priority) 수정 가능 위치
- **상세 페이지(detail.html)**: 조회만 가능. HTML `readonly` 하드코딩. 신규 등록 시 같은 그룹+버전 max+1 자동 부여.
- **목록 화면(index.html)**: ① 인라인 드롭다운(order-select) ② 드래그 핸들(⠿) DnD 두 가지 방법으로만 변경 가능.
- 충돌 시 cascadeShift로 연속 번호 자동으로 뒤로 밀기. GAS LockService로 서버 측 동시성 보호.

### 실시순서(priority) 정렬 규칙
- 1차: 번호 오름차순(1,2,3…). 번호 있는 항목이 항상 위.
- 2차: **둘 다 빈칸이면 티켓번호(XAX2-NNNN 끝 숫자) 오름차순** (`sortByPriority` + `ticketNo` 헬퍼).

### 드래그(핸들) 4케이스 규칙 — `dragend`
드롭 후 DOM에서 드래그 항목의 **이웃**으로 번호영역/빈칸영역 판정
(`inNumberedZone = 바로 뒤가 번호항목 || (맨 끝이고 바로 앞이 번호항목)`):
1. **번호 → 번호 재배치**: 번호 항목들 DOM 순서로 1..n 재번호.
2. **빈칸 → 번호영역**: 드래그 항목이 번호 획득 + 이후 cascade(재번호로 일괄 처리).
3. **번호 → 빈칸영역**: 드래그 항목 번호만 삭제, **원래 번호 자리는 gap으로 유지**(나머지 재번호 안 함).
4. **빈칸 → 빈칸**: 변화 없음(재렌더 시 티켓번호순으로 복귀).
