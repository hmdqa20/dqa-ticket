## 2. 확정된 결정사항 (되돌리지 말 것)

### WW→DQA 내부 리네임 — 하지 않기로 확정 (2026-07-15)
- 표시명은 이미 전부 'DQA'(i18n), WW는 내부 식별자로만 잔존. 분석 결과 8개 파일 ~95곳 + GAS JSON 키(`data.activeWW`) 배포 시차 리스크, HTML id 동적 조립(`'tbody-'+group`) 불일치 시 조용히 깨짐 등 위험 대비 실익 낮음 → **리네임 안 함. 내부 키 activeWW/CSS --ww 등 불변 유지.**

### 실시순서(priority) 규칙 — 코드 대조 검증 완료 (2026-07-15)
- **범위**: 그룹(DQA/MVN)×버전(version_id) 완전 독립. 그룹 판정: 상태(완료→done, 보류/N/A→hold) → assignee('MVN'이면 activeMVN, 아니면 activeWW).
- **정렬(sortByPriority)**: 번호 오름차순 → 빈칸은 티켓번호 오름차순. **빈칸끼리 순서는 사용자가 못 바꿈**(드래그해도 renderAll이 티켓번호순으로 되돌림 — 버그 아님). 완료/보류 섹션은 서버 정렬(status_changed_at desc, 최신 위).
- **드롭다운 변경**: 같은 그룹+버전 내 충돌 시 confirm 후 cascadeShift(연속 번호만 밀고 빈칸에서 중단). 잠긴 항목도 밀림 계산에 포함.
- **드래그 3구간**: A(뒤가 번호)=삽입+cascadeShift / B(번호구역 끝 경계)=max+1만 / C(빈칸구역)=번호 삭제.
- **그룹 이동**(담당자/상태 변경): priority 무조건 초기화. 활성→완료/보류는 서버(updateTicket movingToInactive)도 강제 초기화(이중 안전장치). WW↔MVN은 클라이언트만 초기화(서버 강제 없음 — 비대칭).
- **버전 이동**(moveTicket): priority 무조건 초기화(서버 강제). 번호 유지 옵션 없음 — 새 버전에서 재지정.
- **신규 등록 제안(Rule 1)**: 같은 그룹+버전 max+1 (detail.js getSuggestedPriority). ⚠️ 전체 탭 버그는 TODO 1번.

### 이번 세션 신규 확정 (2026-07-15)
- **완료/보류 이동 직후 로컬 삽입은 unshift(맨 앞)** — 서버의 status_changed_at desc 정렬과 처음부터 일치. push로 되돌리지 말 것.
- **refreshList 버전 가드**: 자동 갱신 응답 도착 시 `vid !== currentVid`면 화면 갱신 스킵(캐시 저장은 vid 키와 짝이 맞아 유지). fetchFreshList의 기존 가드와 한 쌍 — 둘 다 있어야 함.
- **저장 버튼 라벨(등록/저장)은 detail.js `refreshSaveBtnLabel()`이 유일한 결정 지점** — isNewMode에서 항상 유도. 특정 시점에 라벨을 심어두고 나중에 되돌리는 방식 금지(예외 경로 누락 위험). 신규 진입/저장 성공/편집 진입 3곳에서 호출.
- **테스트 모드 배너**: position:fixed + z-index 200(topbar 100보다 위). 실측 높이를 ResizeObserver→`--test-banner-h`로 전달, body padding-top / topbar sticky top / version-sidebar top·max-height가 참조. 운영은 배너 없어 변수 0px — 전부 무영향.

### ⚠️ 터치 드래그 아키텍처 — 3가지 규칙, 하나라도 되돌리면 버그 재발함 (2026-07-11)

1. **`draggable` 속성은 절대 마크업에 하드코딩하지 말 것.** 반드시 `mousedown`이 드래그 핸들 위에서 발생했을 때만 JS로 `row.draggable = true` 동적 부여, 종료 시 `false` 복원. (하드코딩하면 터치 기기에서 네이티브 DnD가 커스텀 터치 핸들러와 경합 → "첫 시도만 되고 이후 안 됨" flaky 버그. `a9216bf`에서 versions.js를 index.js 패턴으로 통일함.)
2. **터치 드롭 위치 판정은 `elementFromPoint`가 아니라 좌표 기반 기하 판정(`resolveTouchDrop`)을 쓸 것.** 손 뗀 지점이 th/여백이면 tr을 못 찾아 드롭이 스킵됨("맨 위로 옮기기" 구조적 실패). `touchmove`(인디케이터)와 `touchend`(실제 이동) 둘 다 같은 함수 사용 — 인디케이터=결과 일치 유지. (`1b2f057`, `807ff14`)
3. **`touchend` 핸들러 맨 앞에 `e.preventDefault()` 유지할 것.** 안 하면 합성(유령) click이 정렬 헤더 등을 눌러 드래그 결과를 덮어씀. (`6cb4884`)

- `setupDragDrop`의 tbody 레벨 리스너는 최초 1회만 부착(중복 방지), 행별 리스너는 tbody 위임 방식.

### GAS_URL — hostname 자동 선택 (2026-07-10~11, 구 규칙 폐기)
- `js/api.js`: `GAS_URL = (location.hostname === 'hmdqa20.github.io') ? PROD : DEV`. **브랜치별로 값 다르게 관리하지 않음** — master/dev의 api.js 항상 동일, 머지 시 GAS_URL 확인 불필요.
- dev 테스트: 로컬(`127.0.0.1:5500`)로 열기(빨간 테스트 배너 뜸). 운영 확인: `hmdqa20.github.io`.

### detail.html 표시모드/편집모드 분리 (2026-07-11)
- 기존 티켓 조회 기본 = 표시(읽기전용) 모드, 잠금은 "수정" 클릭 시에만 획득.

## 4. 주의·보류 사항

- **세션 시작 시 `git pull origin dev` 필수** (회사↔집 누락 사고 방지).
- **커밋만 자동, 푸시는 명시적 요청 시에만.** Code.gs 포함 push 시 재배포 안내 필수.
- **완료/보류&N/A 섹션에는 "티켓이동"(버전 일괄이동) 버튼이 원래 없음** — DQA/MVN 그룹 전용, 버그 아님.
- 티켓 상세 확인버전 입력란·필터 드롭다운에는 CU/CD 색상 점이 안 나옴(input/option 부분 색상 불가) — 목록 테이블 전용, 의도된 범위.
- **터치 드래그 관련 코드(2번 섹션의 3가지 규칙) 리팩터링 시 특히 주의** — 하나라도 되돌리면 flaky 버그 재발 가능성 높음. 수정 전 반드시 실기기(터치) 테스트.
- PowerShell here-string(`@'...'@`)은 중첩 블록 안에서 깨질 수 있음 → 멀티라인 커밋 메시지는 Bash 툴로 커밋.
- Code.gs는 `.gs` 확장자라 `node --check` 불가 → 스크래치에 `.js` 복사본 만들어 검증. CRLF 탓에 큰 블록 Edit 매칭 실패 가능 — 한두 줄 단위로 쪼개서 재시도.
- GAS 실행 6분 제한 — backfillTitleTranslationsForce 티켓 많으면 끊길 수 있음(청크 분할 미구현).
- 헤드리스 브라우저가 로컬에 없음 — CSS/레이아웃 변경은 코드 정합성으로 검증하고 실브라우저 육안 확인은 사용자가 진행.
