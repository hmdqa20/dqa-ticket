## 2. 확정된 결정사항 (되돌리지 말 것)

### 실시순서(priority) 드래그 3구간 판정 — `dragend`(js/index.js)
드롭 후 DOM에서 드래그 항목의 앞/뒤 이웃으로 판정:
- **A. 바로 뒤가 번호항목** → 번호 항목들 DOM 순서로 1..n 전체 재번호(cascade), 빈칸은 그대로
- **B. 바로 뒤는 번호 아님 + 바로 앞은 번호항목(번호구역 끝 경계)** → 드래그 항목만 `max+1`(자기 제외 최댓값) 할당, 원래 자리는 갭 유지, **나머지 항목 불변**
- **C. 양쪽 다 번호 아님(빈칸구역)** → 드래그 항목 번호 삭제(갭 유지), 나머지 불변
- 잠긴 항목(`.locked-row`)도 `rows` 수집에 포함되어 재번호 계산 대상 — 단, 직접 드래그는 여전히 불가(draggable-row 미부여). "핸들 잠금(직접 이동 불가)"과 "시스템에 의한 순서 밀림"은 별개 개념.
- 실시순서 정렬: 번호 있는 항목 오름차순 → 빈칸끼리는 티켓번호 오름차순(`sortByPriority`+`ticketNo`)

### ✅ 잠긴 항목 cascade 규칙 — 확정 완료 (더 이상 보류 아님)
- **규칙**: 다른 항목이 잠긴 항목보다 앞선 순서가 되면(드롭다운/드래그 모두) 잠긴 항목도 동일 그룹 내에서 전체 재정렬(cascade)되어 밀림. 갭이 있으면 채움.
- 잠긴 항목 자체는 사용자가 **직접 드래그 불가**(핸들 잠금)이나, **시스템에 의한 순서 밀림은 발생** — 위 "핸들 잠금 vs 시스템 순서 밀림 별개" 원칙과 일치.
- 드롭다운 경로(`cascadeShift`)는 원래부터 잠금 여부를 안 보고 번호만 보고 밀었으므로 이미 이 규칙대로 동작 중이었음(별도 수정 불필요). 드래그 경로만 버그였고(`.locked-row` 누락) 이번 세션에 수정(`fa35344`)해 **양쪽 경로가 이제 동일 규칙으로 일치**함. → 과거 "옵션 A/B 결정 필요" 논의는 **옵션 B(현행 유지, 자물쇠 번호도 변경)로 확정**.

### 편집 잠금 (locked_at) — 타임아웃 5분 + heartbeat
- **타임아웃 5분**: `js/index.js`의 `LOCK_EXPIRE_MS`(표시용) + `Code.gs`의 `lockTicket()`/`checkLock()`(실제 강제, 두 곳 값 반드시 일치) — 3곳 모두 5분으로 통일됨.
- **heartbeat(2분 주기)**: detail.html 편집 중 `js/detail.js`의 `startHeartbeat(rowId)`가 2분마다 `heartbeat(rowId)` API 호출 → `Code.gs`의 신규 `heartbeat(e)` 함수가 `LOCKED_AT`만 현재 시각으로 갱신(다른 컬럼 불변, LockService 보호). `LOCKED_AT`이 이미 비어있으면(타 세션이 새로 잠금 획득) 갱신 안 하고 무시(`relocked:false`) — 죽은 세션이 남의 잠금 되살리는 사고 방지.
- **정지(`stopHeartbeat`)**는 `releaseLockNow()`/`handleDelete()`/`handleLockBeforeUnload()` 모든 이탈 경로에 연결 — 떠난 세션이 계속 갱신하는 버그 방지.
- **"편집중입니다" 상태 = locked_at 그 자체**, 별도 상태값 없음(과거 언어전환 버그와는 무관한 별개 메커니즘, 이미 분리되어 있음 — detail.js:49 주석 참고).
- 자동저장/무동작(inactivity) 감지 타이머는 **없음, 이번 범위에도 포함 안 함**.

### 이슈명 다국어 번역 — `buildTranslatedTitle(title, targetLang, throttleMs)` (Code.gs)
- **일본어 run 있음** → 그 구간만 개별 번역(기존 방식 그대로, 변경 없음)
- **일본어 run 없음**:
  - 원문이 이미 대상 언어 문자 위주(한글 `[가-힣]` / 베트남어 전용문자) → 원문 그대로(번역 호출 안 함)
  - 그 외(순수 영어 등) → `LanguageApp.translate(title, '', targetLang)`로 **전체 통째 번역**
- `title_ko`/`title_vi` 컬럼엔 순수 번역 결과만 저장. 프론트(`js/index.js`, `js/detail.js`)에서 **일본어 미포함 + 실제 번역됨**이면 `"번역 / 원문"` 슬래시 병기로 표시(목적: "이 티켓이 일본어 없이 등록됐다"는 사실 자체를 알리기 위함). 일본어 포함 시엔 기존처럼 번역 결과만 표시.
- **"i" 아이콘 클릭 시 팝오버는 항상 순수 원문만**(`data-orig`가 `ticket.title` 직접 참조 — 슬래시 병기 문자열이 아님).
- `throttleMs` 인자: 지정 시 `translate` 호출 직후마다 `Utilities.sleep(throttleMs)`. **`addTicket`/`updateTicket` 등 실시간 경로는 인자 안 넘김(기존과 동일, 지연 없음)**. `backfillTitleTranslationsForce`만 `THROTTLE_MS=1000`으로 넘겨 레이트리밋 회피.
- 기존 티켓 소급 적용하려면 GAS 에디터에서 **`backfillTitleTranslationsForce()`** 수동 실행 필요 (완료 로그에 처리/건너뜀/오류 건수 + 실측 소요시간 출력됨). ⚠️ GAS 실행 6분 제한 있음 — 티켓 많으면 중간에 끊길 수 있음(청크 분할은 미구현).

- **커밋만 자동, 푸시는 명시적 요청 시에만** (0번 항목 참고). Code.gs 포함 push 시 재배포 안내 필수.
- **GAS 재배포는 기존 배포 편집 → 새 버전** (새 배포 만들면 URL 바뀌어 앱과 불일치 → 기능 조용히 죽음).
- PowerShell here-string(`@'...'@`)은 `if(){}` 중첩 블록 안에서 깨질 수 있음 → 멀티라인 커밋 메시지는 Bash 툴로 커밋하는 게 안전.
- **Code.gs Edit 시 CRLF로 인해 큰 블록 old_string 매칭이 실패하는 경우가 있음** — 실패하면 작은 단위(한 줄~두 줄)로 쪼개서 재시도할 것.
- Code.gs는 `.gs` 확장자라 `node --check` 불가 → `/tmp`에 `.js` 복사본을 만들어 문법 검증하는 방식 사용 중.
- GAS 실행 시간 제한(약 6분) — `backfillTitleTranslationsForce`가 티켓 수 많으면 중간에 끊길 수 있음(청크 분할 미구현, 필요시 별도 요청).
